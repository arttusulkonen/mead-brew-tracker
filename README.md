# 🍯 Ingria Brewcraft PWA

**Production-Grade System Architecture & Technical Specification**

Профессиональное PWA-приложение для пивоваров и медоваров. Платформа обеспечивает полный цикл производства: от строгого складского учета и конструирования рецептов (с математическим ядром TOSNA 3.0) до "живого" трекинга брожения, ветвления партий (Split-Batch) и AI-аналитики на базе архитектуры RAG.

## 🛠 Технологический стек

- **Frontend:** React 19, TypeScript, Vite, Zustand (State Management), React Router.
- **Backend/DB:** Supabase (PostgreSQL, Auth, RLS, Edge Functions).
- **Math Engine:** `@mead-tracker/math` (изолированный monorepo-пакет).
- **AI & Analytics:** Google Genkit, Gemini (RAG Pipeline), Zod-валидация схем.
- **UI/UX:** Mobile-First, SCSS (БЭМ, CSS-переменные), Recharts (аналитика брожения).
- **Localization:** i18next (автоматизированный пайплайн перевода).

## 🏗 1. Архитектура базы данных (Supabase PostgreSQL)

Мы перешли от NoSQL-документов к строго типизированной реляционной структуре, обеспечивающей ACID-соответствие и высокую безопасность данных через **Row Level Security (RLS)**.

- **`profiles`**: Данные пользователей.
- **`breweries`**: Рабочие пространства (Personal & Shared). Изоляция данных через `owner_id` и `members` (array).
- **`ingredients`**: Мастер-каталог (global) + пользовательские ингредиенты.
- **`inventory`**: Локальный склад (связь `brewery_id` + `ingredient_id`).
- **`recipes`**: Структурированные технологические карты.
- **`brew_sessions`**: Активные варки, поддерживающие транзакционное списание ингредиентов.
- **`fermentation_logs`**: Исторические данные измерений (плотность, pH, температура).

### 🛡 Безопасность и RLS

Все операции записи/чтения защищены политиками RLS, которые проверяют принадлежность пользователя к пивоварне через проверку `auth.uid()` в таблице `breweries`.

## 🧮 2. Математическое ядро (`@mead-tracker/math`)

Модуль вынесен в отдельный workspace-пакет для обеспечения консистентности расчетов между Frontend и Supabase Edge Functions.

### 2.1 Протокол питания дрожжей (TOSNA 3.0)

Динамический расчет органического азота с учетом взвешенного Brix и штамма дрожжей:
`Fermaid-O (g) = ((Brix_weighted * 10 * N_factor) / 50) * V_gal`

### 2.2 Нелинейная модель ABV (Крауч)

`ABV = [(76.08 * (OG - FG)) / (1.775 - OG)] * (FG / 0.794)`

## 🧪 3. Дневник Брожения и Live Feedback Engine

Система `BrewEngine` (Оркестратор) контролирует жизненный цикл `fermenting`.

- **Транзакционность**: В отличие от Firebase, любое списание ингредиентов (списание склада `quantity_on_hand`) теперь происходит через SQL-транзакции, гарантируя целостность остатков.
- **State Machine**: Система не допускает переходов между фазами без фиксации критических параметров (Actual OG, pH).
- **Аналитика**: Наложение `AdvancedFermentationChart` на эталонные кривые прошлых варок (_Golden Batches_) позволяет визуально оценивать динамику в реальном времени.

---

## 🔀 4. Жизненный цикл ветвления (Split-Batch)

Реализован механизм ветвления партий:

1. Родительская варка помечается как `archived/split`.
2. Дочерние варки получают копию всей истории логов до момента `splitTimestamp`.
3. Логирование продолжается в рамках новых сессий с сохранением консистентности данных для каждой емкости.

---

## 🤖 5. Интеграция ИИ: Умный Технолог

Использование **RAG (Retrieval-Augmented Generation)** для генерации рецептов:

1. **Math-to-AI Pipeline**: Клиент рассчитывает параметры (объемы, плотность), ИИ получает "жесткие" данные и не занимается математикой, а лишь интерпретирует стиль.
2. **Валидация**: Ответ ИИ приводится к строгой схеме `Zod`, что исключает ошибки парсинга и галлюцинации.
3. **Guardrails**: Система ограничений в Cloud Functions предотвращает превышение лимитов и защищает от инъекций.

## 📁 6. Структура проекта (Monorepo)

```text
/mead-brew-tracker-pwa
├── ai-backend/            # Supabase Edge Functions (Genkit, AI logic)
├── packages/
│   └── math/              # Изолированное математическое ядро (TOSNA, SG/Brix)
├── public/
│   └── locales/           # i18n JSON файлы
└── src/
    ├── components/        # UI-виджеты (Timeline, IngredientEditor)
    ├── supabase/          # Клиент Supabase, типы, схемы
    ├── pages/             # Маршрутизируемые страницы (Оркестраторы)
    ├── store/             # Zustand State Controllers
    ├── styles/            # SCSS Архитектура
    ├── types/             # Глобальные TypeScript интерфейсы
    └── utils/             # Механика BJCP, константы

```
