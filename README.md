# 🍯 Mead & Brew Tracker PWA

**Production-Grade System Architecture & Technical Specification**

Профессиональное PWA-приложение для пивоваров и медоваров. Платформа обеспечивает полный цикл производства: от складского учета и конструирования рецептов (с математическим ядром TOSNA 3.0) до "живого" трекинга брожения, ветвления партий (Split-Batch) и AI-аналитики.

## 🛠 Технологический стек

- **Frontend:** React 19, TypeScript, Vite, Zustand (State Management), React Router
- **UI/UX:** Mobile-First, SCSS (БЭМ, CSS-переменные, кастомные виджеты FAB/Bottom Sheet), Recharts
- **Backend:** Firebase (Firestore, Auth, Cloud Functions, Hosting)
- **AI & Analytics:** Google Genkit, RAG Pipeline
- **Localization:** i18next (Автоматизированный пайплайн перевода)

## 🏗 1. Архитектура коллекций Firestore (Collection Architecture)

В системе используется гибридный NoSQL-дизайн для обеспечения максимальной производительности, горизонтальной масштабируемости и высокой скорости индексации.

- `/ingredients` — **Глобальный мастер-каталог шаблонов сырья**.
  - `itemId` — Документ: Характеристики конкретного сорта меда, дрожжей, хмеля.
- `/recipes` — **Сохраненные технологические карты (идеальные шаблоны)**.
  - `recipeId` — Документ рецепта со структурированными шагами и фазами.
- `/breweries` — **Изолированные или совместные рабочие пространства пивоварен**.
  - `breweryId` — Документ пивоварни.
    - `/inventory` — **Локальный склад остатков**. Для предотвращения дублирования записей ID документа строго равен `ingredientId` из глобального каталога.
    - `/brew_sessions` — **Активные и исторические варки (живой журнал-регистратор)**.
      - `sessionId` — Документ: статус варки, целевой ABV, фактический объем, расписание TOSNA.
        - `/fermentation_logs` — **Подколлекция периодических измерений брожения**.

### ⚡ Реляционное сопоставление и составные индексы

Для сложной аналитики без падения производительности чтения настроены индексы:

1. **Active Sessions Index** (`sessions`: `breweryId` ASC, `status` ASC, `startedAt` DESC) — мгновенный рендеринг запущенных процессов.
2. **Chronological Log Index** (`fermentation_logs`: `sessionId` ASC, `timestamp` DESC) — последовательное извлечение точек для многоосевых графиков.
3. **Parent-Child Ancestry Index** (`sessions`: `parentSessionId` ASC, `splitTimestamp` ASC) — восстановление древовидной структуры разделенных батчей (Split-Batch).

## 🧮 2. Математическое ядро и расчетные формулы (Calculation Engine)

Модуль `@mead-tracker/math` вынесен в отдельный workspace-пакет для гарантированной консистентности между Frontend и Cloud Functions.

### 2.1 Физика объемов (Volume Physics)

Алгоритм вычисляет общий объем жидкости до начала кипения, учитывая выкипание воды и физический объем растворенного сахара (Плотность мёда $\approx 1.42$ кг/л).
В современном "No-Boil" методе потери на выкипание рассчитываются только для водной матрицы (кипячение хмеля).

### 2.2 Конвертация Плотности и Уровня Алкоголя

- **Brix/SG Конвертер** (с защитой от микро-отрицательных значений):
  `Brix = max(0, 135.997 * SG³ - 630.272 * SG² + 1111.14 * SG - 616.868)`
- **Нелинейная модель Крауча (ABV)**:
  `ABV = [(76.08 * (OG - FG)) / (1.775 - OG)] * (FG / 0.794)`

### 2.3 Протокол питания дрожжей (TOSNA 3.0)

Динамический расчет органического азота с учетом средневзвешенного Brix медовой базы и Nitrogen Demand выбранного штамма дрожжей:
`Fermaid-O (g) = ((Brix_weighted * 10 * N_factor) / 50) * V_gal`

## 🧪 3. Дневник Брожения и Интерактивный Ассистент (Live Feedback)

Компонент `BrewEngine` (Оркестратор) выполняет роль цифрового технолога в фазе `fermenting`.

- **State Machine Workflow**: Переходы статусов строго контролируются событиями (например, ввод `Actual OG` перед стартом). Измерения вносятся через Mobile-First виджет `MeasurementBottomSheet` (Floating Action Button -> Bottom Sheet).
- **Контроль TOSNA**: Если введенная плотность (SG) пересекает отметку $1/3$ Sugar Break, расписание автоматически сжимается (isCompressed: true), а интерфейс выдает Alert на внесение финальной порции питания.
- **Контроль pH**: При падении pH < 3.2 система выдает критический Alert о риске стагнации брожения.
- **Визуализация**: Все логи накладываются на интерактивный `AdvancedFermentationChart` (Recharts) с двухосевым отображением кривой SG и кислотности pH.

## 🔀 4. Жизненный цикл ветвления партий (Split-Batch Lifecycle)

Транзакционный пайплайн для разделения одной партии на несколько емкостей (например, для разной выдержки).

1. Родительская варка замораживается (`isSplit: true`, `batchSizeLiters: 0`).
2. Дочерние сессии глубоко копируют исторический массив `/fermentation_logs` строго до метки времени `splitTimestamp` и наследуют метаданные родителя.
3. Логирование дочерних партий разветвляется и протекает абсолютно изолированно.

## 🤖 5. Интеграция ИИ: Умный Технолог (Google Genkit AI)

Интеграция RAG (Retrieval-Augmented Generation) пайплайна для исключения ИИ-галлюцинаций.

- **Math-to-AI Pipeline**: Приложение проводит точные расчеты физических объемов детерминированным алгоритмом и передает **жесткие числа** в промпт. ИИ запрещено пересчитывать математику — он обязан использовать предоставленные цифры.
- **Локальная База Знаний**: Модель обучается на эталонных рецептах (`recipes.json`) и жестко прописанном технологическом протоколе (`mead_rules.md`).
- **Data Aggregation**: Облачная функция `aggregateRecipeStats` собирает метаданные (средний ABV, общий объем) со всех завершенных варок, формируя аналитическое ядро для Golden Batch.
